──────────┬───────────────────────────────────────────────┬─────────────────────────────┐              
  │          │                     Tool                      │            Skill            │              
  ├──────────┼───────────────────────────────────────────────┼─────────────────────────────┤              
  │ 本质     │ 底层执行能力（如 bash、read、edit、Agent 等） │ 标准操作流程（SOP）         │              
  ├──────────┼───────────────────────────────────────────────┼─────────────────────────────┤              
  │ 粒度     │ 单一、原子性操作                              │ 多步骤、完整的流程指导      │              
  ├──────────┼───────────────────────────────────────────────┼─────────────────────────────┤              
  │ 使用方式 │ 直接调用                                      │ 通过 Agent 或 Tool 组合实现 │              
  ├──────────┼───────────────────────────────────────────────┼─────────────────────────────┤              
  │ 关系     │ Skill 的构成要素                              │ Tool 的组合编排             │
  └──────────┴───────────────────────────────────────────────┴─────────────────────────────┘              
                  
  核心关系                                                                                                
                  
  Skill = Tool 的 SOP（标准操作流程）                                                                     
                  
  - Tool 是"做什么"——单个动作（读文件、执行命令、调用子代理）                                             
  - Skill 是"怎么做"——规定何时用什么 tool、按什么顺序、遵循什么规范
                                                                                                          
  实际例子                                                                                                
                                                                                                          
  以 subagent-driven-development skill 为例：                                                             
                  
  Skill（流程）          Tool（执行）                                                                     
      │                      │                                                                            
      ├─ 读取计划 ──────→  Read tool                                                                      
      ├─ 创建任务清单 ──→  TaskCreate tool                                                                
      ├─ 派发实现子代理 ─→ Agent tool (general-purpose)                                                   
      ├─ 代码审查 ──────→ Agent tool (code-reviewer)                                                      
      └─ 完成任务 ──────→ TaskUpdate tool                                                                 
                                                                                                          
  一句话概括：Tool 是技能，Skill 是把这些技能按最佳实践编排起来的剧本。
