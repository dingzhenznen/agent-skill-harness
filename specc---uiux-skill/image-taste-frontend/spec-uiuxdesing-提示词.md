
  第一段：Spec -> UI 设计文档

  你是资深产品设计师和前端体验设计师。请基于下面这份 spec，产出一份“可直接用于前端实现”的 UI 设计文
  档。

  目标：
  - 不改动 spec 的核心业务目标和功能范围
  - 把需求转成清晰的页面结构、模块层级、交互逻辑和视觉方向
  - 文档要足够具体，方便后续前端代理直接实现

  输出要求：
  1. 产品目标
  2. 目标用户
  3. 页面/路由清单
  4. 每个页面的目标与主要任务
  5. 每个页面的信息架构
  6. 每个页面的模块拆解
  7. 关键交互流程
  8. 组件清单
  9. 状态设计
  10. 响应式设计要点
  11. 视觉风格建议
  12. 文案语气建议
  13. 风险与待确认项

  文档细化要求：
  - 信息架构写清楚页面层级、内容优先级、主次关系
  - 模块拆解写清楚每个区域放什么、为什么放这里
  - 组件清单尽量具体到 navbar、filters、cards、tables、tabs、modals、toasts、pagination 这一级
  - 状态设计必须包含 loading、empty、error、success、disabled、skeleton
  - 响应式部分分别说明桌面端和移动端布局策略
  - 视觉风格建议请用 3-5 个关键词描述，并说明不应该做成什么样
  - 如果 spec 信息不完整，不要擅自编造业务规则；请把缺失项列入“待确认项”

  输出格式：
  用 markdown，按标题分节，内容尽量结构化、具体、可执行。
